## 什么是 xTest Platform？

一个完整的测试序列开发生态系统：

- 深度融合软件工程最佳实践与测试领域专业知识
- 打破单一工具的局限，为测试工程师构建集**流程设计、执行、分析**于一体的全链路武器库
- 重新定义了测试序列开发的工作模式

### 直击行业核心痛点

- 彻底告别传统脚本开发的低效繁琐、技术栈碎片化形成的信息孤岛
- 解决测试异常时问题定位模糊、排查周期长的难题
- 打通团队知识沉淀与传承的壁垒，让测试经验可复用、可迭代

## 测试序列编辑窗口

### 核心作用：自动化测试的“画布”

- 将复杂的测试逻辑转化为直观、易于管理的图形化流程
- 无需编写大量代码，通过拖拽步骤、配置属性，像绘制流程图一样快速构建功能强大的自动化测试用例
- 大幅降低开发门槛

### 编辑窗口界面概览

![编辑窗口界面概览](/editor_overview.png)

1. **顶部功能区**：集成新建、保存、运行、调试等核心操作按钮
2. **序列视图工作区**：核心可视化编排区域，支持拖拽构建测试流程，展示步骤执行逻辑与分支结构
3. **属性配置面板**：选中步骤后实时展示参数配置项，支持自定义输入、断言设置与数据绑定
4. **组件工具箱**：分类收纳所有基础与高级测试步骤（点击、输入、等待、断言等）
5. **变量编辑区**：集中管理当前序列的所有变量，支持定义、赋值和作用域设置
6. **底部状态栏**：显示项目状态、步骤总数与运行环境信息
7. **项目导航区**：快速切换测试用例集，管理项目层级结构

### 编辑窗口核心特性

- **01. 可视化拖拽编程**：摒弃传统代码编写，直接从工具箱拖拽步骤模块，操作直观，上手门槛低
- **02. 全场景内置步骤库**：覆盖API请求、UI交互、数据库操作、文件读写等全类型步骤
- **03. 模块化复用体系**：支持将高频业务逻辑封装为自定义步骤，跨用例、跨项目调用，沉淀测试资产
- **04. 实时错误诊断扫描**：后台实时扫描语法错误、逻辑冲突与参数缺失，错误列表即时展示问题详情与位置
- **05. 高效步骤管理操作**：支持复制、粘贴、撤销、重做、批量排序与删除，跨文件拖拽迁移

## 测试序列调试窗口

### 核心作用：测试执行的“显微镜”

- 可直观监控执行流程，实时查看变量状态与执行日志
- 快速定位异常节点

### 调试窗口界面概览

![调试窗口界面概览](/debug_overview.png)

1. **执行状态实时追踪**：清晰展示每一步的执行进度与状态，可视化进度条与状态标识
2. **变量数据动态监控**：实时刷新关键变量取值，支持查看类型、作用域及变更历史
3. **调用堆栈深度分析**：完整展示函数调用链路与层级关系，快速定位问题调用节点
4. **全量输出日志查看**：整合系统日志、操作日志与报错信息，支持按级别筛选与关键词检索

### 调试窗口核心特性

- **01. 灵活断点配置**：支持普通断点、条件断点与命中次数断点，按需精准暂停
- **02. 精细化单步控制**：提供步过、步入、步出，逐层深入校验执行流程
- **03. 实时变量监视**：实时显示变量值，支持运行中手动修改，快速验证不同参数下的结果
- **04. 堆栈与断言定位**：清晰展示调用堆栈路径，断言失败时自动跳转至问题代码行
- **05. 全流程状态可视化**：整合线程状态、执行日志与错误高亮，提升排错效率

## xTestCopilot — 您的AI测试副驾

**深度集成的智能伙伴**：将AI注入测试全流程，自动生成测试序列、智能诊断失败原因；具备动态知识库，感知项目上下文，回答各类测试相关问题。

![xTestCopilot 智能助手对话框界面](/xcopilot_dialog.png)

*（xTestCopilot 对话框界面，清晰展示了能力列表、使用前提与三种工作模式入口）*

### 核心能力全景

不仅是对话机器人，更是IDE原生的智能协作者。从需求解析到用例生成，从故障定位到知识沉淀，全方位覆盖测试开发生命周期。

### 核心AI能力

- **01. 需求驱动测试生成**：通过自然语言描述业务需求，Copilot 自动解析并生成结构化、可执行的完整测试序列框架
- **02. 智能诊断失败根因**：自动抓取测试失败日志与运行上下文，结合模型推理分析异常原因，精准定位问题节点并提供修复建议
- **03. 上下文感知知识库**：无缝接入项目技术文档与历史测试数据，理解业务场景上下文，提供贴合项目实际的解答

### xTestCopilot 三种工作模式

- **Ask（问答模式）**：快速获取针对性信息与解答，如“解释当前序列文件的内容？”得到清晰解答与要点说明
- **Agent（代理模式）**：扮演任务执行者，直接在IDE环境中响应指令并执行操作，如“帮我修复这个序列的问题”自动完成修复
- **Plan（规划模式）**：专为复杂任务设计，先将需求拆解为详细执行计划，确认逻辑可行后再逐步落地，如“根据要求生成测试序列”时先分析需求、规划流程再执行

三种模式从基础问答到自动执行，再到复杂规划，形成完整能力闭环，极大提升工作灵活性与效率。
